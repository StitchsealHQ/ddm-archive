# 변경 이력

## 2026-07-21

수집 범위를 '재규정한 동대문'(도심형 복합 산업 클러스터) 기준으로 넓히고, 유튜브 영상을 새 소스로 추가했다. 데이터 누적 구조와 대시보드 UI도 함께 정비했다.

### 수집 범위 확장 — 완구/트렌드·인근상권 (`1512755`)

검색 쿼리를 7종에서 17종으로 늘리고 카테고리 2종을 신설했다. 신규 쿼리는 실제 API 결과를 받아본 뒤 품질로 선별했다(단독 '말랑이'·'키캡'·'장충동'은 노이즈가 많아 제외).

| 카테고리 | 쿼리 |
|---|---|
| 완구/트렌드 | 동대문 완구거리, 창신동 완구, 말랑이 도매, 동대문 키링 |
| 인근상권 | 광장시장, 경동시장, 청계천 상가, 을지로 상권, 힙당동, 명동 쇼핑 |

- **AI 관련성 프롬프트 재작성** — 기존 프롬프트는 "종로·남대문 등 타 상권 단독 기사는 무관"으로 판정해 신규 쿼리 수집분을 도로 지웠다. 중심 허브(동대문 패션타운·DDP)와 종로·청계·을지로·글로벌링크(명동)·북부·남부 라인을 모두 수집 범위로 명시하고, 오탐 제외 규칙(경동나비엔, 개혁신당, 키보드 키캡)을 추가했다.
- **이미지 앵커 필터** — AI 관련성 판정은 뉴스·블로그만 검사해서, 광역 쿼리로 들어온 무관 이미지가 무검사 통과하던 구멍이 있었다. 신규 카테고리의 이미지는 제목에 지역·품목 관련어가 있을 때만 유지하도록 규칙을 추가했다.
- **검색 트렌드 5번째 그룹** — 네이버 데이터랩에 `말랑이·왁뿌볼` 추가. 최근 4주 상대지수 100 → 80 → 62로, 동대문 관련 키워드(맛집 ~11, 동대문 ~8)의 6~8배였다.

첫 운영 실행에서 무관 자료 306건이 필터로 제거됐다.

### 유튜브 영상 수집 (`0f31082`)

`youtube.py` 신규. YouTube Data API v3로 17개 쿼리의 최근 14일 영상을 검색하고, 제목·채널명·설명란을 GitHub Models(gpt-4.1)에 넣어 관련성 판정과 요약을 함께 생성한다. `영상` 종류 카드로 게시하며 썸네일과 원본 링크를 함께 제공한다.

- 자막(스크립트) 추출은 유튜브가 클라우드 IP를 차단해 크론에서 불안정하므로 **설명란 기반**으로 결정했다. 설명란 원문은 저장하지 않고 요약만 남긴다(기존 저작권 원칙과 동일).
- AI 호출이 실패하면 무판정으로 유입시키고 다음날 `ai_dedup.py`가 재검사하도록 이중화했다.
- 쿼터는 하루 약 1,700유닛으로 무료 한도 10,000의 17% 수준이다.

첫 운영 실행에서 122건 중 22건이 AI 판정으로 제외되고 100건이 게시됐으며, 전부 요약을 확보했다.

### 연도별 데이터 분리 (`34862b0`)

`data.json`이 매일 누적되며 무한정 커지는 구조를 정리했다.

- `archive.py` 신규 — 수집일(`added`) 기준 90일이 지난 자료를 `docs/data-<수집연도>.json`으로 이동하고, `data.json`의 `archives` 필드에 파일 목록과 건수를 기록한다. 라이브 파일은 항상 최근 90일치만 유지된다.
- 사이트는 기본으로 라이브 파일만 불러오고, 검색·북마크·딥링크·"지난 자료 불러오기" 버튼에서만 아카이브를 추가로 불러온다. 오래된 공유 링크도 자동으로 아카이브에서 찾아 열린다.
- **함께 고친 버그** — `collect.py`가 `data.json`을 저장할 때 `generated_at`과 `items`만 다시 써서 `trend`·`briefing` 필드를 매일 날리고 있었다. 그 탓에 월요일에만 생성되어야 할 주간 브리핑이 매일 재생성되고 있었다.

### 대시보드 UI 개선 (`2fd109b`, `212fd45`)

- **유형 필터를 칩으로 승격** — 같은 날짜 항목이 배열 순서대로 정렬되는 탓에, 하루 수십 건씩 벌크로 들어오는 네이버 블로그가 뉴스·영상을 첫 화면 밖으로 밀어냈다(영상 100건이 첫 48장에 한 장도 없었다). 유형을 드롭다운에서 칩으로 올려 건수와 함께 노출하고, 최신순의 동일 날짜 타이브레이커를 유형 우선순위(뉴스 → 영상 → 블로그 → 이미지)로 지정했다. 첫 화면 구성이 뉴스 8 → 12장, 영상 0 → 4장으로 바뀌었다. 'AI 요약 우선' 정렬도 추가했다.
- **상단 위젯 접기** — 검색 트렌드를 접어 기본은 제목 줄만 보이게 했다. 모바일에서는 주간 브리핑을 3건까지만 노출한다. 첫 카드까지의 스크롤이 데스크톱 1,222 → 745px, 모바일 1,654 → 1,229px로 줄었다.
- **이미지 수집량 축소** — 네이버 이미지는 발행일이 없어 목록 최하단에 고정되고 AI 요약도 불가한데 전체의 35%를 차지했다. 쿼리당 30 → 10건으로 줄였다. 기존 499건은 수집일 기준 90일 뒤 아카이브로 자동 이동한다.

### 운영 메모

- 저장소 Secret에 `YOUTUBE_API_KEY`를 등록했다. 기존 `NAVER_CLIENT_ID`/`NAVER_CLIENT_SECRET`와 함께 사용한다.
- 워크플로 단계 순서: collect → youtube → ai_dedup → summarize → **archive** → trend → briefing → commit.
- 첫 실제 아카이빙은 기존 자료의 `added` 백필 시점 기준 90일 뒤인 10월 중순부터 시작된다.
